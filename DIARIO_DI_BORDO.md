### 2026-03-28 – 🚀 MIGRAZIONE COMPLETA: da Render a GitHub Pages + Supabase

#### ❌ Problema con Render
Il backend FastAPI su **Render** (piano gratuito) presentava un problema critico:
- Il server si **addormentava** dopo 15 minuti di inattività
- Al risveglio impiegava 30-60 secondi con richieste che andavano in timeout
- In molti casi il server **non si riavviava correttamente**, rendendo l'app inutilizzabile
- Non adatto per un progetto che deve essere condiviso con amici e sempre disponibile

#### ✅ Nuova Architettura (Always-On, Gratuita, Multi-Utente)

La nuova architettura elimina completamente il server backend:

| Componente | Prima (Render) | Dopo (GitHub Pages) |
|-----------|---------------|-------------------|
| **Frontend** | HTML servito da FastAPI | HTML/CSS/JS statico su GitHub Pages |
| **Backend** | Python FastAPI su Render | ❌ Eliminato! Logica nel browser |
| **Database** | SQLite locale sul server | PostgreSQL su Supabase (cloud) |
| **Autenticazione** | Username/password custom | Email/password via Supabase Auth |
| **Analisi ritardi** | Python (`analisi_ritardi.py`) | JavaScript (`analisi.js`) nel browser |
| **Storico estrazioni** | Download via server Python | File nel repo + GitHub Action automatica |
| **Costo** | Gratuito ma instabile | Gratuito e sempre attivo |
| **Multi-utente** | ❌ Problematico | ✅ Nativo con isolamento dati (RLS) |

##### 2. Schema Database PostgreSQL (Supabase)
Creato tramite SQL Editor di Supabase, 3 tabelle:

**Tabella `strategies`** – Strategie di gioco per utente:
```sql
CREATE TABLE strategies (
    id BIGSERIAL PRIMARY KEY,
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    ruota TEXT NOT NULL,         -- Codice ruota: BA, MI, FI, ecc.
    ambo_1 INTEGER NOT NULL,    -- Primo numero dell'ambo
    ambo_2 INTEGER NOT NULL,    -- Secondo numero dell'ambo
    estratto INTEGER NOT NULL,  -- Numero per gioco estratto
    attiva BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT now()
);
```

**Tabella `user_plans`** – Piano economico per utente:
```sql
CREATE TABLE user_plans (
    id BIGSERIAL PRIMARY KEY,
    user_id UUID NOT NULL UNIQUE REFERENCES auth.users(id) ON DELETE CASCADE,
    budget_per_estrazione REAL DEFAULT 6,    -- € per estrazione
    estrazioni_pianificate INTEGER DEFAULT 10, -- Durata piano
    data_inizio TEXT,
    created_at TIMESTAMPTZ DEFAULT now()
);
```

**Tabella `extractions_played`** – Registro estrazioni giocate:
```sql
CREATE TABLE extractions_played (
    id BIGSERIAL PRIMARY KEY,
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    data_estrazione TEXT NOT NULL,
    giocata BOOLEAN DEFAULT false,
    speso REAL DEFAULT 0,
    UNIQUE(user_id, data_estrazione)
);
```

**Row Level Security (RLS)**: Attivata su tutte le tabelle. Ogni utente può vedere e modificare SOLO i propri dati:
```sql
CREATE POLICY "Users manage own strategies" ON strategies 
    FOR ALL USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id);
```

##### 3. Conversione Codice Python → JavaScript

- `downloadStorico()` – Scarica lo storico dal repository GitHub (raw file)
